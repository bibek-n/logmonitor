Implement real agent uninstall on Linux; document removal for both OSes

Linux UninstallService() was a stub that always errored ("not supported on
Linux"). It now actually stops and disables the systemd unit, removes the
unit file, reloads systemd, deletes /etc/logmonitor-agent (config plus
log-shipping state), and removes the binary itself in one command: sudo
logmonitor-agent uninstall.

Windows UninstallService() now also removes the ProgramData config
directory after stopping and unregistering the service (it already worked,
this just makes the cleanup complete). The .exe itself is left for the
admin to delete by hand, since Windows locks a running executable.

Both Download Agent pages (endpoint agents and servers) and the agent
README now document the uninstall command for each OS.
